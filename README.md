# Threads投稿スタジオ

店舗向けSNSスレッド作成支援Webアプリケーション。テンプレートを選択し、必要情報を入力するだけで、効果的なThreads投稿スレッドを簡単に作成できます。

## 主な機能

### 1. テンプレート選択
4種類の最適化されたテンプレートから選択可能：
- **共感→問題提起→解決策→行動喚起**: 読者の共感を得て行動を促す王道パターン
- **FAQ型**: よくある質問に答える形式で信頼を構築
- **体験談型**: 実際のお客様の声や事例を通じて効果を伝える
- **チェックリスト型**: 自己診断できるチェックリスト形式で関心を引き出す

### 2. 入力フォーム
テンプレートに応じた必要情報を入力：
- 店舗名、対象、悩み、ベネフィット、CTA、注意書き等
- トーン選択（丁寧/カジュアル/専門寄り）
- ポスト数と1ポストあたりの最大文字数設定

### 3. スレッド生成
- ルールベースの生成ロジック
- 広告規制・誇大表現を回避する安全フィルタ搭載
- トーンに応じた語尾調整

### 4. エディタ
- ドラッグ&ドロップでポストを並び替え
- ポスト単位で編集・追加・削除
- 文字数カウンタと上限超え警告
- リアルタイムプレビュー表示

### 5. ライブラリ
- 保存したプロジェクトの一覧表示
- 検索・タグ・並び替え機能
- プロジェクトの作成日時・更新日時表示

### 6. 書き出し
複数の形式で書き出し可能：
- **テキスト形式**: シンプルなテキストファイル
- **JSON形式**: プログラムで扱いやすい形式
- **CSV形式**: 表計算ソフトで開ける形式

各形式でコピーとダウンロードに対応。

### 7. AI補助機能
- **ハッシュタグ候補生成**: 入力内容から関連するハッシュタグを自動生成
- **フック案生成**: 読者の注意を引く1行目のフレーズ案を生成

## 技術スタック

- **フロントエンド**: React 19 + TypeScript
- **ルーティング**: Wouter
- **UI**: shadcn/ui + Tailwind CSS 4
- **ドラッグ&ドロップ**: @hello-pangea/dnd
- **データ永続化**: LocalStorage
- **デザイン**: エレガントなカラーパレット（紫系）+ Playfair Display + Inter

## 使い方

### 1. テンプレートを選択
ホーム画面で4つのテンプレートから最適なものを選択します。

### 2. 情報を入力
スタジオ画面で必要な情報を入力し、トーンと文字数制限を設定します。

### 3. スレッドを生成
「スレッドを生成」ボタンをクリックすると、エディタ画面に遷移します。

### 4. 編集
エディタ画面でポストを編集・並び替え・追加・削除します。右側のプレビューで確認しながら調整できます。

### 5. 保存
「保存」ボタンでプロジェクトをLocalStorageに保存します。

### 6. 書き出し
「書き出し」ボタンから書き出し画面に遷移し、テキスト・JSON・CSV形式で書き出せます。

### 7. ライブラリ
保存したプロジェクトはライブラリ画面から検索・閲覧・再編集できます。

## テンプレート追加方法

新しいテンプレートを追加するには、`shared/templates.ts`の`TEMPLATES`配列に新しいテンプレートオブジェクトを追加します。

```typescript
{
  id: "new-template",
  name: "新しいテンプレート",
  description: "テンプレートの説明",
  icon: "🎯",
  requiredFields: [
    { key: "field1", label: "フィールド1", placeholder: "例：...", required: true },
    // ...
  ],
  outputRules: [
    { role: "hook", contentTemplate: "{{field1}}を使った内容" },
    // ...
  ],
}
```

## AI接続ポイント

現在はルールベースの生成ロジックですが、以下のポイントでAI（LLM）に接続可能です：

### 1. スレッド生成
`shared/generator.ts`の`generateThread`関数を拡張し、LLM APIを呼び出すように変更します。

```typescript
// 例: invokeLLMを使った実装
import { invokeLLM } from "../server/_core/llm";

export async function generateThreadWithAI(
  template: Template,
  inputs: ProjectInputs,
  options: GenerateOptions
): Promise<Post[]> {
  const prompt = `以下の情報を元に、${template.name}形式のスレッドを生成してください...`;
  
  const response = await invokeLLM({
    messages: [
      { role: "system", content: "あなたはSNS投稿の専門家です" },
      { role: "user", content: prompt },
    ],
  });
  
  // レスポンスをパースしてPost[]に変換
  // ...
}
```

### 2. ハッシュタグ生成
`shared/generator.ts`の`generateHashtags`関数をAI接続に変更します。

### 3. フック案生成
`shared/generator.ts`の`generateHooks`関数をAI接続に変更します。

### 実装のヒント
- サーバーサイド（tRPC procedure）でLLM呼び出しを行う
- フロントエンドからはtRPCのmutationで呼び出す
- ストリーミングレスポンスにも対応可能

## 安全フィルタ

広告規制・誇大表現を回避するため、以下のパターンを自動置換します：

- `治る/治す/治療` → `整える`
- `必ず/絶対/100%` → 削除
- `根本改善` → `根本からサポート`
- `完治` → `サポート`
- `最高/最強/No.1` → `おすすめ`
- `劇的/驚異的` → `心地よい`

新しいパターンを追加するには、`shared/generator.ts`の`UNSAFE_PATTERNS`配列に追加します。
